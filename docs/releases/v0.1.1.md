# v0.1.1 — Agent / MCP scoring refresh

Public release date: **2026-07-16**

This release keeps HealthAI Audit current for the AI tools small practices are actually evaluating: agentic schedulers, MCP/tool-broker connectors, and unsupervised automation — not only ambient scribes and RAG chatbots.

## What changed

- **Agent domain scoring** now reviews:
  - MCP / tool-broker server inventories (`mcp_servers`)
  - autonomous mode (`autonomous_mode`)
  - network egress posture (`network_egress`)
  - high-impact tools such as browser, webhook, SMS, calendar write, and MCP
- **Critical flags** for unsupervised autonomous agents and unapproved MCP connections
- **Sample inventory** adds a fictional **Front Desk MCP Scheduling Agent** that scores Critical for demo/teaching
- Inventory **template** includes the new optional fields (backward compatible with v0.1.0 inventories)
- Method string bumped to `HealthAI Audit deterministic scoring v0.1.1`

## Optional inventory fields (v0.1.1)

| Field | Purpose | Example values |
| --- | --- | --- |
| `mcp_servers` | Named MCP/tool-broker servers the agent can reach | `["practice-calendar", "sms-gateway"]` |
| `autonomous_mode` | Whether the agent can act without a human per step | `no` / `partial` / `yes` |
| `network_egress` | Outbound network posture | `none` / `limited` / `internet` |

Existing inventories without these fields still score. Missing agent mode for tool-using systems is treated as an unknown gap.
