# v0.4.0 — Automated policy packs (`healthai-audit run`)

Public release date: **2026-07-18**

Customers should not choose policy packs or assemble packet steps by hand.

## One command

```bash
healthai-audit run path/to/inventory.json
```

Automatically:

1. Safety-check the inventory (fail closed)
2. **Detect** practice policy pack + overlays
3. Score all tools
4. Apply pack-specific rules
5. Decide block / restrict / approve*
6. Write owner packet + kit bridge + `RUN_SUMMARY.md`

Optional:

```bash
healthai-audit run inventory.json --out reports/client-acme
healthai-audit detect-pack inventory.json   # show pack only
```

## Auto pack selection

| Signal | Result |
| --- | --- |
| `practice_profile.type` or name/tool keywords (`dental`, `therapy`, …) | Primary pack |
| `states` length > 1 or `multi_state: true` | `multi_state` overlay |
| `msp_managed: true` or review_owner contains MSP | `msp_managed` overlay |

Primary packs: `general_small`, `dental_small`, `behavioral_health`, `specialty_clinic`

Pack rules (examples):

- `HA-PACK-DENTAL-001` imaging AI without BAA
- `HA-PACK-BH-001` behavioral sensitive workflow without clinician review
- `HA-PACK-MS-001` multi-state clinical tool without state-policy review
- `HA-PACK-MSP-001` MSP agent tools without customer disable switch

No `--pack` flag is required. Optional `practice_profile` only improves detection; inference works from practice name + tools.

## Customer inventory shape (minimal)

```json
{
  "practice": "Maple Grove Dental",
  "review_owner": "Office Manager",
  "review_date": "2026-07-18",
  "practice_profile": {
    "type": "dental",
    "states": ["MO", "IL"],
    "msp_managed": true
  },
  "tools": [ { "name": "...", "vendor": "...", "workflow": "...", "...": "..." } ]
}
```

If `practice_profile` is omitted, the tool still auto-selects from signals.
