# v0.2.0 — Decision workbench (10x path)

Public release date: **2026-07-16**

v0.1.x scored inventories. **v0.2.0 turns scores into owner decisions** and
fails closed when an inventory looks unsafe.

## Why this is the 10x step

A risk score alone does not change practice behavior. What changes behavior:

1. **block / restrict / approve** decisions a manager can act on
2. **stable rule IDs** (`HA-BAA-001`, `HA-MCP-001`, …) for handoffs and re-audits
3. **owner packets** (markdown + CSV + JSON) ready for MSP / kit import
4. **fail-closed safety** so secrets and free-text PHI risk never enter reports

## What shipped

### Safety (default fail-closed)

- File size and shape limits
- Secret/PHI pattern scan (keys, SSN-shaped values, private keys, JWTs, …)
- Reject free-text fields (`notes`, `prompt`, `transcript`, …)
- Strip raw inventory `source` from reports by default
- `--include-source` only after redaction of high-risk fields
- `--allow-unsafe` escape hatch (not recommended)

### Decisions

- Per-tool decision: `block` | `restrict` | `approve_with_conditions` | `approve`
- Portfolio decision rolled up from worst tool
- Rule catalog with owners
- Sorted action queue for MSP/owner handoff

### Packet export

```bash
healthai-audit packet samples/sample_inventory.json --out reports/decision-packet
```

Writes:

- `owner-decision-packet.md`
- `action-queue.csv`
- `decisions.json`
- `vendor-followups.md`

### CLI

| Command | Purpose |
| --- | --- |
| `score` | Score + decisions; optional `--packet-dir` |
| `packet` | Decision packet only |
| `safety-check` | Scan without scoring |
| `validate` | Schema-ish warnings + safety |
| `template` | Inventory / questionnaire / policy starters |

## Boundaries

Still triage support only. Not legal, clinical, HIPAA, FDA, or certification advice.
Do not put PHI, credentials, private URLs, contracts, logs, patient details, or
incident details into inventory files.
