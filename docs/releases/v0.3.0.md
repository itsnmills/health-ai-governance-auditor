## 1. Evidence refs (approve requires proof)

Inventories may include `evidence_refs` on each tool:

```json
"evidence_refs": [
  {
    "id": "EVID-BAA-001",
    "kind": "baa",
    "path": "evidence/vendor-baa.pdf",
    "sha256": "optional-64-hex",
    "reviewed_on": "2026-06-01",
    "expires_on": "2027-06-01",
    "covers_rules": ["HA-BAA-001"]
  }
]
```

Rules:

- **Reference-only** — path / id / hash / date / kind. No notes, contract text, or PHI.
- Relative local paths only (no `..`, no URL schemes).
- PHI-touching tools cannot receive unconditional **`approve`** without a non-expired covering ref (`baa`, `policy`, `soc2`, `training_opt_out`, or `clinician_signoff`).
- New rules: `HA-EVID-001` (missing), `HA-EVID-002` (expired).

## 2. Kit bridge (Small Practice Security Kit import)

```bash
healthai-audit kit-export samples/sample_inventory.json --out reports/kit-bridge
# or included under packet:
healthai-audit packet samples/sample_inventory.json --out reports/decision-packet
```

Writes:

| Artifact | Role |
| --- | --- |
| `ai-workflow-review.md` | Kit table: allowed / restricted / prohibited |
| `handoff-actions.csv` | Owner/MSP AI handoff rows (`stage_id=ai_phi_review`) |
| `kit-import-manifest.json` | Schema `velari.kit_bridge.v1` |

Decision mapping: `approve→allowed`, `restrict|approve_with_conditions→restricted`, `block→prohibited`.

## 3. Diff mode (closed vs regressed rules)

```bash
healthai-audit diff samples/sample_inventory.json samples/sample_inventory_remediated.json
healthai-audit diff before/decisions.json after/decisions.json --format json
```

Reports:

- closed rule IDs (good)
- new rule IDs (regressions / newly detected)
- decision improvements vs regressions
- evidence status changes
- tools added/removed

Sample pair: `samples/sample_inventory.json` → `samples/sample_inventory_remediated.json`.
